# データ要素市場供給側管理システム - フロントエンド

**バックエンドのリポジトリはこちら:** [data-element-backend](https://github.com/yexca/data-element-backend)

## 概要

本プロジェクトは、データ要素市場の供給側管理システムのための、応答性の高いユーザーインターフェースを提供します。Vue.jsエコシステム（Vue Router, Vuex）を基盤に構築されており、ユーザー、管理者、そして一般訪問者という異なるロールに対して、最適化されたUI/UXを提供します。

特に、グローバルな利用を想定した**多言語対応（日中英）**機能を`vue-i18n`を用いて実装している点が特徴です。

## 🌟 主な機能

* **リアクティブUI**: Vue.jsによるコンポーネントベースのUI開発。
* **グローバル状態管理**: Vuexを使用し、ユーザーのログイン状態や言語設定などのアプリケーション全体の状態を一元管理。
* **多言語対応**: `vue-i18n`を導入し、日本語、中国語、英語の動的な言語切り替えに対応。
* **クライアントサイドルーティング**: Vue Routerによる、シームレスな画面遷移を実現するシングルページアプリケーション(SPA)。
* **動的なUI分離**:
  * **公共ポータル**: 未ログインユーザー向けのデータ製品検索・閲覧機能。
  * **ユーザーダッシュボード**: 個人/企業ユーザー向けの登録、ログイン、および自身のデータ製品の管理（CURD）機能。
  * **管理ダッシュボード**: 従業員/管理者向けの、全ユーザー情報、全データ製品、カテゴリ情報などを管理する包括的なバックオフィス機能。
* **HTTPクライアント**: Axiosをカプセル化し、リクエスト/レスポンスインターセプターを実装。JWTトークンの自動添付や、グローバルなエラーハンドリングを処理。
* **フォームバリデーション**: ElementUIの検証ルールと連携し、登録・ログイン・データ入力フォームにおけるリアルタイムの入力値検証を実装。

## 🛠 使用技術

| カテゴリ         | 技術                        |
| :--------------- | :-------------------------- |
| コア             | Vue.js                      |
| 状態管理         | Vuex                        |
| ルーティング     | Vue Router                  |
| UIライブラリ     | ElementUI                   |
| HTTPクライアント | Axios                       |
| 国際化(i18n)     | Vue-i18n                    |
| ビルド・デプロイ | Node.js, Nginx (Docker経由) |

## 起動方法

1. **リポジトリをクローン:**

   ```bash
   git clone https://github.com/yexca/data-element-frontend.git
   cd data-element-frontend
   ```

2. **依存関係のインストール:**

   ```bash
   npm install
   ```

3. **開発サーバーの起動 (HMR):**

   ```bash
   npm run serve
   ```

4. **プロダクションビルド:**

   ```bash
   npm run build
   ```

**注意:** 本フロントエンドは、バックエンドのAPIサーバーが起動していることを前提としています。`vue.config.js` のプロキシ設定を確認し、必要に応じてバックエンドサーバーのアドレスを調整してください。
